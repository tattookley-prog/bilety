# Билет №3 — NFS

## Что требуется по заданию
Поднять NFS-сервер на HQ-SRV, экспортировать каталог и подключить его на HQ-CLI в `/mnt/nfs` с автомонтированием.

## Теория
NFS (Network File System) — сетевая файловая система: клиент работает с удалённой папкой почти как с локальной.
Модель «сервер/клиент»: сервер публикует экспорт, клиент монтирует его по сети.
Главный файл сервера — `/etc/exports`: в нём путь и правила доступа для сетей/хостов.
Частые опции: `rw` (чтение/запись), `sync` (подтверждать запись после записи на диск), `no_subtree_check` (упрощает проверки подкаталогов), `no_root_squash` (root клиента остаётся root на сервере — использовать осторожно).
`rpcbind` и сервисы NFS должны быть запущены для корректной работы.
Для постоянного подключения клиент добавляет запись в `/etc/fstab`, часто с `_netdev`, чтобы система ждала сеть.

Для защиты полезно объяснить не только команды, но и логику потока запроса/данных:
что является точкой входа, где хранится состояние, и каким шагом подтверждается корректность.
Хорошая практика ответа: сначала назвать роль сервиса, затем ключевой конфиг, затем проверочную команду.
Так преподавателю видно, что вы понимаете причину настройки, а не только повторяете команды.

## Ключевые файлы и пути конфигурации
- `/etc/exports`
- `/etc/fstab` (на клиенте)
- Точка экспорта (например, `/raid5/nfs`)
- Точка монтирования клиента `/mnt/nfs`

## Основные команды
- `exportfs -ra` — перечитать экспорты.
- `exportfs -v` — показать активные экспорты.
- `showmount -e <server>` — что экспортируется.
- `mount -t nfs ...` — ручное монтирование.
- `mount | grep nfs` / `df -h` — факт монтирования.
