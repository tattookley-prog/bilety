# Билет №8 — Moodle

## Что требуется по заданию
Развернуть Moodle на HQ-SRV: веб-сервер Apache, PHP-FPM, MariaDB, нужные PHP-расширения и доступ к приложению по HTTP.

## Теория
Moodle — LMS (Learning Management System), веб-приложение на PHP с базой данных.
Типовой стек: Apache (или другой веб-сервер) + PHP-FPM + MariaDB.
PHP-FPM обрабатывает PHP-код, Apache отдаёт HTTP и проксирует PHP-запросы в FPM.
Для Moodle нужны расширения PHP, включая `mysqli` для работы с MariaDB.
Параметр `max_input_vars` часто увеличивают для корректной работы больших форм и настроек Moodle.
В лабораторной схеме Moodle может работать на нестандартном порту (например, `8081`) через VirtualHost/Alias.
Установка возможна через веб-мастер или через CLI-инструменты.

Для защиты полезно объяснить не только команды, но и логику потока запроса/данных:
что является точкой входа, где хранится состояние, и каким шагом подтверждается корректность.
Хорошая практика ответа: сначала назвать роль сервиса, затем ключевой конфиг, затем проверочную команду.
Так преподавателю видно, что вы понимаете причину настройки, а не только повторяете команды.

## Ключевые файлы и пути конфигурации
- Конфиг Apache VirtualHost (порт `8081`)
- Конфиг PHP (`php.ini`, FPM pool)
- Каталог Moodle (web root)
- БД `moodledb`

## Основные команды
- `systemctl is-active httpd2/apache2` — статус веб-сервера.
- `systemctl is-active php-fpm` — статус PHP-FPM.
- `php -m | grep -i mysqli` — наличие расширения.
- `curl -I http://localhost:8081/moodle/` — HTTP-проверка.
- `mysql -e "SHOW DATABASES;"` — наличие БД.
