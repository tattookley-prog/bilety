# Билет №11 — sudo

## Что требуется по заданию
На HQ-CLI настроить sudo-права для доменной группы `hq`: разрешить только ограниченный набор команд (`cat`, `grep`, `id`) и запретить остальное.

## Теория
`sudo` позволяет запускать команды с повышенными привилегиями по контролируемым правилам.
Основной синтаксис находится в `/etc/sudoers`, но практичнее использовать отдельные файлы в `/etc/sudoers.d/`.
`Cmnd_Alias` группирует команды в логический набор для удобной выдачи прав.
`NOPASSWD` позволяет выполнять разрешённые команды без запроса пароля.
Правила можно выдавать группам (включая доменные `%group`) — это упрощает администрирование.
`visudo` обязателен для проверки синтаксиса: ошибка в sudoers может заблокировать админ-доступ.
В доменной среде PAM/SSSD и `pam_mkhomedir` помогают корректному входу и созданию домашнего каталога.

Для защиты полезно объяснить не только команды, но и логику потока запроса/данных:
что является точкой входа, где хранится состояние, и каким шагом подтверждается корректность.
Хорошая практика ответа: сначала назвать роль сервиса, затем ключевой конфиг, затем проверочную команду.
Так преподавателю видно, что вы понимаете причину настройки, а не только повторяете команды.
На защите полезно показать границу полномочий: разрешённые команды работают, а неразрешённые отклоняются.

## Ключевые файлы и пути конфигурации
- `/etc/sudoers`
- `/etc/sudoers.d/hq`
- PAM-конфиги с `pam_mkhomedir` (при доменной авторизации)

## Основные команды
- `visudo -c -f /etc/sudoers.d/hq` — проверка файла правил.
- `sudo -l -U user1hq` — какие команды реально разрешены.
- `su - user1hq` + тестовые `sudo` команды — демонстрация ограничений.
