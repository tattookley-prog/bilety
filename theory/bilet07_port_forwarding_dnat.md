# Билет №7 — Port forwarding / DNAT

## Что требуется по заданию
Настроить проброс портов на роутерах (HQ-RTR/BR-RTR): DNAT для SSH (2024) и, для BR-RTR, HTTP (80 → BR-SRV:8080).

## Теория
NAT меняет адрес/порт пакета при прохождении через маршрутизатор.
DNAT (Destination NAT) меняет адрес назначения входящего трафика, SNAT — исходящий адрес.
MASQUERADE — частный случай SNAT, удобный для динамических внешних адресов.
В iptables DNAT обычно делается в `nat/PREROUTING`, затем трафик должен пройти через цепочку `FORWARD`.
После маршрутизации часто добавляют `nat/POSTROUTING` (SNAT/MASQUERADE), чтобы обратный путь работал корректно.
Обязательное условие маршрутизатора — `net.ipv4.ip_forward=1`.
Правила нужно сохранять, иначе после перезагрузки они исчезнут.

Для защиты полезно объяснить не только команды, но и логику потока запроса/данных:
что является точкой входа, где хранится состояние, и каким шагом подтверждается корректность.
Хорошая практика ответа: сначала назвать роль сервиса, затем ключевой конфиг, затем проверочную команду.
Так преподавателю видно, что вы понимаете причину настройки, а не только повторяете команды.

## Ключевые файлы и пути конфигурации
- Параметр ядра `net.ipv4.ip_forward`
- Таблица `nat` и цепочки `PREROUTING`, `POSTROUTING`
- Цепочка `FORWARD`

## Основные команды
- `sysctl net.ipv4.ip_forward` — включена ли маршрутизация.
- `iptables -t nat -L PREROUTING -n --line-numbers` — DNAT-правила.
- `iptables -L FORWARD -n` — разрешения форвардинга.
- `iptables -t nat -L POSTROUTING -n` — SNAT/MASQUERADE.

## Частые вопросы преподавателя / на что обратить внимание

- Почему без `ip_forward=1` проброс не заработает.
